# LearnSphere Backend Architecture: The Modular Monolith

## Overview
LearnSphere is built using the **Modular Monolith** architectural pattern. This means that while it is a single application (not microservices), it is strictly organized into layers (Controllers, Services, Repositories, Models) to ensure it can be easily maintained or even split into microservices in the future.

## 🏁 Starting Point: Program.cs
The absolute entry point of the server is [Program.cs](file:///c:/Users/londh/Downloads/Practice-main/test/LearnSphereBackend-master/Program.cs).
1. **Host Building**: It sets up the Kestrel web server.
2. **Service Registration (DI)**: It registers every Repository and Service so they can be "Injected" into controllers.
3. **Middleware Pipeline**: It defines the "Order" of operations (Security -> Routing -> Authorization -> Controllers).

## 🔄 The Request/Response Cycle
When a user clicks "Login" or "View Course", the following chain happens:

### 1. The Request (Handled by Controllers)
The request hits an **API Controller** (e.g., `CoursesController.cs`).
- **Input Validation**: The `[ApiController]` and `DataTransferObjects (DTOs)` ensure the data is safe and correctly formatted.
- **Security Check**: `[Authorize]` attributes verify the student's JWT token.

### 2. The Logic (Handled by Services)
If the task is complex (like uploading a video), the Controller calls a **Service** (e.g., `FileUploadService.cs`).
- Services contain the "Business Rules" (e.g., "Files must be less than 500MB").

### 3. The Data (Handled by Repositories)
To talk to SQL Server, the Controller (or Service) calls a **Repository** (e.g., `CourseRepository.cs`).
- Repositories use **Entity Framework Core** and `AppDbContext.cs` to run clean, optimized SQL queries.

### 4. The Response (Generated by DTOs)
The server never returns raw database "Models". Instead, it maps the data to a **Response DTO** (e.g., `CourseResponseDto.cs`).
- This ensures only "Public" data is sent back, keeping secrets (like password hashes) safe.

## 🛡️ Security & Integrity
- **Authentication**: Uses **JWT (JSON Web Tokens)**. Once a user logs in, the server gives them a "Key" that they must present for every future request.
- **Data Integrity**: Enforced by `AppDbContext` via unique indexes and cascading deletes.
- **UTC Sync**: All dates are normalized to UTC globally to prevent time-zone bugs between students across the world.

## 🚀 Why not Microservices? (The Defense)
For the current scale of LearnSphere, a Modular Monolith is **superior** because:
1. **Zero Network Latency**: Communications happen in memory, not over slow network calls between services.
2. **Simplified Deployment**: One server to manage, not twenty.
3. **Strong Consistency**: Transactions are handled by a single database, avoiding "Distributed Transaction" headaches.
4. **Future Proof**: Because we use interfaces and DI (Dependency Injection), we can easily slice a component (like the Quiz engine) into its own microservice later if needed.
