# Documentation: SafeHtmlRenderer.jsx

## Overview
`SafeHtmlRenderer.jsx` is a robust component used to display HTML generated by the `VisualEditor`.

## Detailed Explanation
This component is significantly more advanced than `SafeMarkdown`. It uses the browser's `DOMParser` logic (via a temporary `div`) to physically walk through a piece of HTML and "Scrub" it. It maintains a "Whitelist" of allowed tags (v, `p`, `ul`, `h1`, etc.) and removes anything that isn't on the list.

## Why it is used
It is the "Shield" for the platform. It allows instructors to have rich formatting while ensuring students are 100% safe from malicious code injection.

## Interview Questions
1. **How does this component handle "Unknown" tags?**
   *Answer: It replaces them with a text node of their content. If someone tries to use a `<form>` tag, the form is removed, but the text inside the form remains.*
2. **Explain the attribute filtering logic.**
   *Answer: It only allows specific attributes like `href` and `class`. For links, it specifically checks that the `href` doesn't start with `javascript:`, which is a common vector for XSS.*
3. **Why are the styles defined inside the component?**
   *Answer: To provide "Scoped" styling. By targeting `.safe-html-renderer h1`, it ensures that course content headlines look consistent regardless of what global CSS is loaded.*
